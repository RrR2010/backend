# TASK: Protect and Refactor Critical Controllers

---

## DESCRIPTION

- Summary (1 sentence): Update the controllers that currently rely only on authentication so they use explicit authorization metadata and tenant-aware access.
- Steps (max 5):
  1. Audit the main controllers for missing public/protected metadata.
  2. Apply the new authorization decorators to the critical routes.
  3. Refactor controllers that currently mix tenant resolution with authorization decisions.
  4. Remove direct access assumptions that are not backed by policy.
  5. Verify admin, tenant, membership, and user routes all follow the new contract.
- Implementation hints:
  - Controllers should declare intent, not encode business authorization rules inline.
  - Keep controller changes shallow when possible; move complex checks into policies and use cases.
  - Controllers must NOT trust tenantId from request input for tenant users
  - Controllers must delegate all scoping logic to query layer or service layer
