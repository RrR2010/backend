# TASK: Implement Authorization Decorators

---

## DESCRIPTION

- Summary (1 sentence): Add explicit decorators to mark public routes and request authorization metadata on protected routes.
- Steps (max 5):
  1. Implement a public-route decorator that marks endpoints as intentionally unauthenticated.
  2. Implement an authorization decorator for action/subject declarations on protected endpoints.
  3. Support optional metadata for scope and ABAC conditions when needed.
  4. Document the decorator usage pattern for controllers.
  5. Export the decorators from a single entry point.
- Implementation hints:
  - The decorators should be declarative metadata only; they should not perform policy evaluation.
  - Keep route intent obvious: public routes must be explicit, and protected routes must declare authorization intent.
  - Avoid overloading a single decorator with too many optional behaviors.
