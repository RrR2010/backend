# TASK: Normalize Authorization Domain Model

---

## DESCRIPTION

- Summary (1 sentence): Consolidate the canonical authorization vocabulary for roles, scopes, actions, subjects, and permission specs.
- Steps (max 5):
  1. Review the current enums and permission types under `backend/src/core/domain`.
  2. Define the canonical model for `Action`, `Subject`, platform scope, tenant scope, and permission spec shapes.
  3. Align the model with the current auth/session payloads and membership model.
  4. Remove or isolate ambiguous naming that could be confused between platform and tenant authority.
  5. Expose the final canonical types through a small shared export surface.
- Implementation hints:
  - Keep the MVP model small; do not introduce DB-backed policy concepts yet.
  - Make role names and subject names explicit and stable, because they will be used by decorators and guards.
  - Separate request context data from resource attributes so ABAC conditions stay readable.
