# TASK: Implement Request-Scoped Ability Factory

---

## DESCRIPTION

- Summary (1 sentence): Build CASL abilities per request using the authenticated actor, tenant context, membership roles, and resource attributes.
- Steps (max 5):
  1. Define the request context shape consumed by the ability factory.
  2. Build the ability per request from platform roles, tenant roles, tenant context, and request scope.
  3. Support a small set of canonical actions and subjects.
  4. Include minimal ABAC conditions for tenant boundaries and ownership.
  5. Expose the result through the Authorization module for use by guards and services.
- Implementation hints:
  - Keep the ability factory deterministic and side-effect free.
  - Treat tenant membership as a first-class part of the authorization context.
  - Avoid leaking business logic into the factory; it should only translate context into permissions.
- Ability must encode scope:
  - Tenant users → restricted by tenantId
  - Platform users → unrestricted or conditionally restricted
