# TASK: Startup Authorization Validation

---

## DESCRIPTION

- Summary (1 sentence):
  Validate at application startup that every route explicitly declares authorization metadata (`@Public` or `@RequireAbility`), enforcing a fail-closed model at development time.

- Steps (max 5):
  1. Use NestJS metadata exploration (e.g., `DiscoveryService`, `MetadataScanner`, `Reflector`) to scan all controllers and route handlers.
  2. For each route, check for presence of either `@Public()` or `@RequireAbility(...)` metadata.
  3. Collect all routes missing authorization metadata.
  4. If any invalid routes are found, throw an error during application bootstrap (fail startup).
  5. Output a clear, structured error message listing controller, method, and route path.

## DEFINITION OF DONE (DoD)

- [ ] All controllers and route handlers are scanned at application startup
- [ ] Routes without `@Public()` or `@RequireAbility(...)` are detected
- [ ] Application fails to start if any route lacks authorization metadata
- [ ] Error output clearly identifies controller name, method name, and route path
- [ ] Validation runs automatically during bootstrap (no manual trigger required)
- [ ] Public and protected routes are both accepted patterns

## 10. NOTES

- This task enforces fail-closed behavior at development time, not only at runtime.
- Prevents accidental exposure of routes due to missing decorators.
- Must not introduce significant startup overhead.
- Should be implemented as a lightweight validation step during bootstrap.
- Prefer explicit failure over warnings (security-first approach).
