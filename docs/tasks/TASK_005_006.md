# TASK: Implement Platform Scope Policies

---

## DESCRIPTION

- Summary (1 sentence): Define the platform-scoped authorization rules for internal users who can operate across tenants.
- Steps (max 5):
  1. Map platform roles to the actions they can perform across the platform.
  2. Define which subjects are globally visible to platform users.
  3. Allow explicit support/admin operations while keeping the rule set minimal.
  4. Prevent platform rules from accidentally granting tenant-bound data access without explicit scope.
  5. Document the intended use of each platform role.
- Implementation hints:
  - Platform access should be explicit and narrowly scoped to support/administration use cases.
  - Do not allow platform scope to bypass tenant checks unless the route intentionally requires that.
