# TASK: Create Authorization Module Structure

---

## DESCRIPTION

- Summary (1 sentence): Introduce a dedicated backend module for authorization providers, policy definitions, and ability construction.
- Steps (max 5):
  1. Create a new Authorization module folder structure with clear boundaries for domain, application, infra, and interface concerns.
  2. Add providers for policy registry, ability factory, and authorization metadata resolution.
  3. Wire the module into the root application module without changing behavior yet.
  4. Export only the minimal providers needed by controllers and guards.
  5. Keep the module independent from business modules except for typed dependencies.
- Implementation hints:
  - Model this as a shared cross-cutting module, not as logic embedded in each feature module.
  - Keep provider names explicit so later refactors can locate the authorization path quickly.
  - Do not move business rules into the module; only centralize authorization concerns.
