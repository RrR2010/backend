# TASK: Integration Tests for Leakage Scenarios

---

## DESCRIPTION

- Summary (1 sentence): Validate the full request pipeline against public routes, protected routes, and cross-tenant leakage cases.
- Steps (max 5):
  1. Build integration tests for public routes that should remain accessible.
  2. Test protected routes with valid and invalid authorization metadata.
  3. Verify cross-tenant requests are rejected end to end.
  4. Exercise sensitive read paths to ensure query-layer scoping is active.
  5. Confirm the API returns `403 Forbidden` for authorization failures.
- Implementation hints:
  - Treat this task as a security regression suite.
  - Include at least one test that would have leaked data before the refactor.
