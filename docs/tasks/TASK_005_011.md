# TASK: Implement Minimal ABAC Ownership Rules

---

## DESCRIPTION

- Summary (1 sentence): Add a small set of attribute-based checks for resource ownership, creator ownership, and tenant association.
- Steps (max 5):
  1. Identify the resources that genuinely need ownership-based rules.
  2. Add the minimum attributes required to express those rules.
  3. Model ownership and tenant association as CASL conditions where appropriate.
  4. Avoid turning every entity into an ABAC subject.
  5. Document which resources use ABAC and why.
- Implementation hints:
  - ABAC should only appear where RBAC is insufficient for the MVP.
  - Start with tenantId and ownerId/createdBy; add more attributes only if a real use case requires them.
- ABAC usage must be minimal and limited to:
  - tenant boundary checks
  - ownership checks (`ownerId`, `createdBy`)
- New attributes must NOT be introduced unless strictly required.
