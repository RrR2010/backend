# TASK: Refactor Login UseCase to Return Unified Auth Result

---

## DESCRIPTION

Refactor the login use case so it returns a single, stable auth result for both platform and tenant users.

- Summary (1 sentence):
  Remove heuristic branching from the login path and return a deterministic auth result containing the user, available contexts, and next-step hint for the frontend.
- Steps (max 5):
  1. Refactor `LoginUseCase` to return a unified auth result object
  2. Expose available contexts explicitly (`platform` and/or tenants)
  3. Remove hidden assumptions based on role arrays or null tenant handling
  4. Keep tenant lookup logic isolated from token issuance
  5. Preserve backward compatibility only where needed for tests during migration
