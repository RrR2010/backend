# TASK: Logout Should Revoke Session in Database

---

## DESCRIPTION

The logout endpoint only clears cookies but does not revoke the session in the database. This causes users to see duplicate sessions after logging out and logging back in.

- Summary (1 sentence):
  Modify the logout endpoint to revoke the session in the database before clearing cookies.
- Steps (max 5):
  1. Modify logout endpoint to extract refresh token from cookie
  2. Validate refresh token to get session ID
  3. Call RefreshTokenService.revokeRefreshToken() to mark session as revoked
  4. Clear all session cookies
  5. Handle errors gracefully (clear cookies even if revocation fails)
