### Implementation Notes:

nest-casl provides (per documentation):
- `CaslModule.forRoot<T>()` - global config with typed roles
- `superuserRole` - role with unrestricted access
- `getUserFromRequest` - extracts user (default: request.user)
- `AccessGuard` - replaces custom PermissionsGuard
- `@UseAbility` - replaces custom @CheckPermissions
- `createCallerFactory` - for service-level checks

Our JWT token structure (AuthTokenPayload):
```typescript
interface AuthTokenPayload {
  sub: string;       // user ID
  tenantId?: string; // tenant context (optional)
}
```

Integration: request.user is already set by passport-jwt strategy, nest-casl can use it directly.
