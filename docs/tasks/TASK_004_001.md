# TASK: Add Session/RefreshToken Model to Prisma Schema

## DESCRIPTION

Add database models to track sessions and refresh tokens for session hardening.

- Summary (1 sentence):
  Extend the Prisma schema with Session and RefreshToken models to enable session tracking, refresh token rotation, and revocation capabilities.
- Steps (max 5):
  1. Add `Session` model with fields: id, userId, refreshTokenHash, deviceInfo, ipAddress, createdAt, expiresAt, revokedAt, isRevoked
  2. Add `RefreshToken` model (or integrate into Session) for rotation tracking
  3. Add relation between Session and User
  4. Run Prisma migration to update database schema
