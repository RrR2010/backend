# TASK: Implement Authorization Logging

---

## DESCRIPTION

- Summary (1 sentence): Log denied authorization decisions with enough context to support debugging, audit, and security analysis.
- Steps (max 5):
  1. Define the minimum log payload for authorization denials.
  2. Add logging to the guard or shared authorization service on denied access.
  3. Include actor identity, route, action, subject, and tenant context when available.
  4. Avoid logging sensitive secrets or tokens.
  5. Make the logs easy to filter in development and production.
- Implementation hints:
  - Logs should be useful for incident response without exposing sensitive data.
  - Keep the log structure stable so later observability tooling can parse it.
- Define a structured log format for authorization denials including:
  - actorId
  - scope (platform | tenant)
  - tenantId
  - action
  - subject
  - route
  - reason
