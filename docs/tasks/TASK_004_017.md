# TASK: Define Auth Context Contract and Scope Claims

---

## DESCRIPTION

Define a stable authentication context contract for platform and tenant users.

- Summary (1 sentence):
  Introduce a single auth context contract so the backend and frontend can consistently understand whether the current session is platform-scoped or tenant-scoped.
- Steps (max 5):
  1. Define a unified auth payload contract for `scope`, `sub`, `tenantId?`, and session identifiers
  2. Keep `scope` informational for routing/context resolution, not as a source of authorization truth
  3. Define the response contract for `/auth/login` and `/auth/select-tenant`
  4. Define the response contract for frontend redirect decisions
  5. Update DTOs and shared auth types if needed
