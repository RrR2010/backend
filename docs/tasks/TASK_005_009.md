# TASK: Refactor or Remove Unsafe Repository Methods

---

## DESCRIPTION

- Summary (1 sentence): Fix repository interfaces and implementations that currently allow unscoped reads or inconsistent contracts.
- Steps (max 5):
  1. Audit repository interfaces and implementation mismatches.
  2. Align method signatures with the required authorization scope.
  3. Remove or deprecate methods that can return unscoped data without tenant context.
  4. Update dependent use cases to use the new scoped methods.
  5. Verify that the repository contract reflects the actual security model.
- Implementation hints:
  - Do not keep ambiguous methods just for backward compatibility if they create leakage risk.
  - This task should make missing scope impossible or visibly incorrect at compile time.
  - Repository methods MUST require tenantId for tenant-scoped entities
  - Global (unscoped) queries must not exist for tenant-bound data
