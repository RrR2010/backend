# TASK: Implement Tenant Scope Policies

---

## DESCRIPTION

- Summary (1 sentence): Define the tenant-scoped rules for customer users acting only within their membership context.
- Steps (max 5):
  1. Map tenant roles to the actions they can perform inside a tenant.
  2. Define tenant-bound subjects and the minimum resource ownership rules.
  3. Use tenant membership as the source of truth for tenant access.
  4. Ensure no tenant-scoped rule can cross tenant boundaries.
  5. Document the expected behavior for tenant administrators versus tenant users.
- Implementation hints:
  - Tenant scope should always be constrained by `tenantId` or equivalent membership context.
  - Do not rely on client-provided tenant IDs without validating membership and request context.
  - All tenant permissions MUST include tenantId condition
