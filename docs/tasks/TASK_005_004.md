# TASK: Implement Global Authorization Guard (Fail-Closed)

---

## DESCRIPTION

- Summary (1 sentence): Add a global guard that enforces route metadata, denies missing authorization intent, and blocks unauthorized access.
- Steps (max 5):
  1. Create a global authorization guard that reads public/protected metadata from the route.
  2. Fail closed when the route is neither public nor explicitly protected.
  3. Allow explicit public routes without requiring ability evaluation.
  4. Evaluate the request ability for protected routes and return `403 Forbidden` on failure.
  5. Log denied access attempts with enough context for debugging and audit.
- Implementation hints:
  - This guard should only decide access; it should not build policies itself.
  - Use the guard to enforce the contract that every route must be explicit.
  - Missing metadata must be treated as a security error, not as a default allow.
